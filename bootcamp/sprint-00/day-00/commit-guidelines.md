# Git Commit Convention

We use Conventional Commits.

---

## Types

### feat
New feature

feat: add promise creation endpoint

---

### fix
Bug fix

fix: correct promise deadline validation

---

### chore
Maintenance

chore: update docker setup

---

### refactor
Code refactoring without behavior change

refactor: improve promise service structure

---

### test
Adding tests

test: add unit tests for promise service

---

### docs
Documentation

docs: update API description

---

## Rules

- One commit = one logical change
- No "final fix", "update", "tmp"
- Commit messages must be descriptive
